Build a polished enterprise web application prototype for “Bid Intelligence” for Cummins Aftermarket.

TECH STACK
- React + Vite + TypeScript
- Tailwind CSS
- Component-driven architecture
- Single-page app with role-based views
- No backend required
- Use realistic mock data only
- No URL routing required; use tab/button navigation within the app
- Make the application feel like a real SaaS platform suitable for a client demo

PRIMARY PRODUCT GOAL
This platform supports the bid lifecycle across multiple user roles:
- Scouts identify and filter opportunities from multiple bid portals
- Admins (Daily Bids team) process incoming bids using 3 intake patterns and generate draft responses
- Account Executives (AEs) review summarized bid packages and make final decisions

IMPORTANT UX CONTEXT
The platform should feel like one integrated system, not separate apps.
All roles use the same underlying bid/opportunity data model, but each role sees different screens, actions, and navigation options.

ROLES TO SUPPORT

1. Admin (Primary Persona / Daily Bids Team)
Purpose:
- Main operational user for the MVP
- Handles 3 primary intake patterns:
  a) Email Body intake
  b) Excel Attachment intake
  c) External URL / public portal link intake
- Reviews extracted parts and quantities
- Checks AI-derived insights
- Generates or edits bid responses
- Moves bids through qualification and response preparation

2. Scout
Purpose:
- Monitors portals and discovers opportunities
- Filters signal from noise
- Reviews AI relevance scoring
- Shortlists and hands off relevant opportunities for Admin processing

3. Account Executive (AE)
Purpose:
- Reviews qualified bids
- Sees summarized, executive-friendly views instead of raw ingestion details
- Approves, requests changes, or rejects bids
- Reviews risks, pricing summary, and recommended next actions

HIGH-LEVEL USER FLOWS

FLOW 1: Admin / Daily Bids Team
1. Intake bid from one of 3 patterns
2. Review extracted metadata and line items
3. Validate AI extraction confidence and unmatched items
4. Review pricing / inventory / lead time summary
5. Open Response Workbench
6. Generate, edit, and export response draft
7. Mark ready for AE review or submit mock response

FLOW 2: Scout
1. Review portal health and new opportunities
2. Run portal scans / refreshes / upload exports
3. Filter opportunities by relevance score and customer fit
4. Open opportunity detail and review AI rationale
5. Shortlist / hand off to Admin

FLOW 3: AE
1. Open approval queue
2. Review executive bid summary
3. Review summarized pricing, risks, and recommendation
4. Approve / Request Changes / Reject
5. Track post-submission outcomes
